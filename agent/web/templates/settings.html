Telemetry ingest

{% if not ingest_enabled %}

INGEST_TOKEN is not set in .env. The POST /ingest endpoint will return 503 until you set it. Pick any opaque string.

{% else %}

Ingest is active. Your live systems can POST events to http://127.0.0.1:8765/ingest and they'll show up on /events grouped by source.

Example from a shell script or Python cron:

curl -X POST http://127.0.0.1:8765/ingest \
  -H "Content-Type: application/json" \
  -H "X-Ingest-Token: $INGEST_TOKEN" \
  -d '{
    "source": "blitzpicks",
    "event_type": "weekly_accuracy",
    "id": "2026-04-22-nba-weekly",
    "title": "Weekly NBA prop accuracy hit 84.1% (+1.9% WoW)",
    "body": "2,847 props graded. XGB-only baseline 82.3% vs ensemble 84.1%. Largest gap on 3PM and blocks — LSTM is catching fatigue patterns the tree models miss."
  }'

id is the dedup key — same id + same source = same event, silently ignored. Omit it and a content hash is used instead.
